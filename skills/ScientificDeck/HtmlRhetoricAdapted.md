# Rhetoric of Decks — Adapted for HTML

## Three Laws (unchanged)
1. **Beauty = Clarity** — Every DOM element justifies its presence
2. **Cognitive Load is the Enemy** — One idea per viewport-height slide
3. **Slides Support Speech** — Visual anchors, not articles

## HTML Adaptations
- One idea = one `<section class="slide">` at `100dvh`
- Assertive `<h1>` titles as claims, not labels
- CSS classes for semantic depth: `.slide--title`, `.slide--content`, `.slide--data`, `.slide--figure`, `.slide--section`, `.slide--quote`
- Chart.js replaces ggplot, KaTeX replaces LaTeX math, Mermaid replaces TikZ
- Same audience calibration: External (polish) / Internal (rough) / Students (scaffolding)

## Key Principle
"KI-Fluency ist augmentativ" — the presenter sets the standard, slides support the argument.
